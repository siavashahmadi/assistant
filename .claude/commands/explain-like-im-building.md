Sia wants to understand **$ARGUMENTS** by building something.

## Step 1: Load Context
Read:
- `memory/me.md` — stack, interests, learning style

## Your Job

Don't explain the concept with theory and diagrams. Instead, design a **small, buildable project** (completable in 1-3 sessions) that forces him to understand it through implementation.

## Rules
- Use his stack where possible: React, TypeScript, Python, Flask, Supabase, PostgreSQL
- The project should be **interesting enough to actually want to build** — not a todo app unless the concept demands it
- Start with the **minimum viable version**, then list stretch goals
- Each step should teach one aspect of the concept

## Output Format

### The Concept in 30 Seconds
2-3 sentences max. Just enough to start building.

### The Project: [Creative Name]
One paragraph describing what it does and why it's a good vehicle for learning $ARGUMENTS.

### Build Steps

Step-by-step implementation order. Each step:

**Step N: [What you're building]**
- What to implement
- **What this teaches about $ARGUMENTS:** (1-2 sentences connecting the code to the concept)
- Code hints if the step is non-obvious

### Stretch Goals
2-3 ways to deepen understanding after the core is working.

### "You'll Know You Get It When..."
One sentence describing the aha moment — what understanding looks like.

## After generating

Ask if Sia wants to:
1. Save this to `outputs/build-[concept-slug].md`
2. Start building it right now (scaffold on ~/Desktop/)
