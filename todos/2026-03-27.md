## EnGen Application Answers

**Walk us through a recent feature or project you shipped. What did you own end-to-end, and what were the hardest technical decisions you made?**

At Joulea, I owned the full IoT data pipeline end-to-end — from sensor ingestion through RabbitMQ, normalization, fault detection, and dual storage routing to RDS and S3. It processes ~10M data points per day across 200+ commercial buildings.

The hardest technical decision was around storage architecture. We were hitting cost and query performance walls with everything living in RDS Aurora. I had to figure out what data needed to stay in the relational database for real-time fault detection APIs versus what could be offloaded to S3 for historical analytics. I ended up designing a migration using AWS Glue that routed older data to S3 and built Athena-backed APIs for the analytics layer. That cut our storage costs by 57% without sacrificing query performance on the real-time side.

The other hard call was on the normalization layer. IoT sensor data comes in from different vendors with inconsistent formats and labeling. I deployed a generative-AI labeling workflow using the OpenAI API to classify unstructured data automatically — it increased labeling speed by 67% and maintained 90% accuracy. The decision to use LLMs for this instead of a rule-based system was a bet, but it paid off because the vendor inconsistencies were too unpredictable for static rules.

**Describe how you currently use AI tools (e.g. Cursor, Claude Code) in your day-to-day development workflow. What works well, and where do you still rely on writing code by hand?**

Claude Code is basically my co-pilot at this point. I use it for scaffolding new features, writing tests, refactoring messy code, and especially for rubber-ducking architectural decisions — I'll describe what I'm trying to build and go back and forth until the approach feels right. It's really good at boilerplate, migrations, and catching things I'd miss in review.

Where I still write by hand: anything that touches core business logic or tricky state management. AI tools are great at generating code that looks right but doesn't account for the weird edge cases you only know about from living in the codebase. Same with debugging — I'll use AI to help narrow down where something's breaking, but the actual root cause analysis usually comes from me reading the code and understanding the system. Also anything security-sensitive or involving data handling constraints — I want to know exactly what's happening line by line.

**What excites you about EnGen's mission to support English learners with workforce appropriate instruction?**

My parents came to the US from Iran when I was two. They spoke Farsi at home, and I watched them navigate a world where their English was functional but never quite comfortable — especially in professional settings. The gap between conversational English and the kind of English you need to actually advance in a career is real, and most language tools don't address that gap at all. EnGen focusing specifically on workforce-appropriate instruction hits close to home. Building the technology that helps people like my parents close that gap is genuinely meaningful to me.
